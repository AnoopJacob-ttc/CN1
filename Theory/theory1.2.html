CISCO IOS ACCESS METHODS

METHOD DESCRIPTION
CONSOLE This is a physical management port that provides out-of-band access to a Cisco device. Out-of-band access refers to access via a dedicated management channel that is used for device maintenance purposes only. The advantage of using a console port is that the device is accessible even if no networking services are configured, such as performing the initial configuration. A computer running terminal emulation software and a special console cable to connect to the device are required for a console connection.
SECURE SHELL (SSH) SSH is an in-band and recommended method for remotely establishing a secure CLI connection, through a virtual interface, over a network. Unlike a console connection, SSH connections require active networking services on the device, including an active interface configured with an address. Most versions of Cisco IOS include an SSH server and an SSH client that can be used to establish SSH sessions with other devices
TELNET Telnet is an insecure, in-band method of remotely establishing a CLI session, through a virtual interface, over a network. Unlike SSH, Telnet does not provide a secure, encrypted connection and should only be used in a lab environment. User authentication, passwords, and commands are sent over the network in plaintext. The best practice is to use SSH instead of Telnet. Cisco IOS includes both a Telnet server and Telnet client.

NAVIGATE BETWEEN DIFFERENT IOS MODES

User-EXEC Mode

Prompt: Switch>

Purpose: Basic monitoring; cannot make configuration changes.

Next Step: Type enable to enter Privileged EXEC mode.

Privileged EXEC Mode

Prompt: Switch#

Purpose: Full command access, view and modify configurations.

Enter: enable (from User-EXEC)

Exit: disable (back to User-EXEC)

Global Configuration Mode

Prompt: Switch(config)#

Purpose: Configure system-wide settings.

Enter: configure terminal or conf t

Exit: end or Ctrl + Z

Special Configuration Modes

Interface Mode: Switch(config-if)# → interface fa0/1

Routing Protocol Mode: Switch(config-router)# → router ospf 1

Exit: exit → back to Global Configuration

Switch>				//User Exec Mode
Switch>enable
Switch#				// Privileged Exec Mode
Switch# configure terminal
Switch(config)#			//Global Configuration Mode
Switch(config)#interface fa0/1
Switch(config-if)#		//Interface Mode (Specific Configuration Mode)

BASIC CISCO DEVICE CONFIGURATION

SET THE CLOCK

Switch>enable
Switch#clock set 10:30:15 01 January 2026
Switch#show clock

HOST NAME

Switch>enable
Switch#configure terminal
Switch(config)#hostname S1
S1(config)#

BANNER MOTD

S1(config)#banner motd "Unauthorized access is strictly prohibited"

DISABLE DNS LOOKUP

Switch#configure terminal
Switch(config)#no ip domain-lookup

CONFIGURE DOMAIN NAME

Switch(config)#ip domain-name nttftrg.com

ENABLE PASSWORD

Switch(config)#enable password cisco

ENABLE SECRET

Switch(config)#enable secret itsasecret

CONFIGURE LINE PASSWORDS

Switch(config)#line console 0
Switch(config-line)#password cisco
Switch(config-line)#login

Switch(config-line)#line vty 0 15
Switch(config-line)#password cisco
Switch(config-line)#login
Switch(config-line)#exit
Switch(config)#

ENCRYPT ALL PLAIN TEXT PASSWORDS

Switch(config)#service password-encryption

LOGGING SYNCHRONOUS

Switch(config)#line console 0
Switch(config-line)#logging synchronous

Switch(config-line)#line vty 0 15
Switch(config-line)#logging synchronous

LOGIN ATTEMPT LIMITATION SETTINGS - EXEC TIMEOUT

Switch(config-line)#line console 0
Switch(config-line)#exec-timeout 5

Switch(config-line)#line vty 0 15
Switch(config-line)#exec-timeout 5
Switch(config-line)#end
Switch#

SAVE CONFIGURATION FILES TO NVRAM

S1(config)#exit
S1#copy running-config startup-config

CONFIGURE IP ADDRESS MANUALLY ON AN END DEVICE

Follow these steps to manually configure the IP address on a Windows computer:

  1. Open Network Settings
    • Go to the "Start" menu and click on "Control Panel".
    • In the "Control Panel", go to "Network and Sharing Center".
    • Click on "Change adapter settings" on the left pane.
    • Right-click on your active network connection (Ethernet or Wi-Fi) and select "Properties".
  2. Select TCP/IPv4
    • In the "Properties" window, select Internet Protocol Version 4 (TCP/IPv4) and click on the "Properties" button.
  3. Assign IP Address
    • In the new window, choose the option: Use the following IP address.
    • Enter the desired IP address (for example, 192.168.1.10), subnet mask (255.255.255.0), and default gateway (192.168.1.1) for the device.
  4. Set DNS Servers
    • If required, enter the DNS servers. For example, you can use Google's DNS servers:
    • Preferred DNS server: 8.8.8.8
    • Alternate DNS server: 8.8.4.4
  5. Save and Apply
    • Click OK to save the settings, and then close the remaining windows.
    • Your PC is now manually configured with the static IP address.

Verification:

To verify the configuration:

  • Open Command Prompt and type the following command to check the IP configuration:
    • ipconfig
  • Verify that the IP address, subnet mask, and default gateway appear correctly.
