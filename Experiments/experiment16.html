Experiment – 16

CONNECT TWO NETWORKS USING A CISCO ROUTER (IPV4 AND IPV6)

← Back to Experiment List ← Back to Main Page

AIM

  - Connect two switches to the router
  - Connect one PC to each switch
  - Configure basic settings on all devices
  - Assign different IPv4 and IPv6 networks to each LAN
  - Configure router interfaces with IPv4 and IPv6 addresses
  - Assign IP addresses to both PCs
  - Implement SSH on the router for secure access
  - Test connectivity between PCs using the ping command
  - Verify directly connected networks

R1 CONFIGURATION

Router>
Router>enable
Router#conf t
Router(config)#hostname R1
R1(config)#enable secret class
R1(config)#service password-encryption
R1(config)#banner motd "Warning! Authorized access only"
R1(config)#no ip domain-lookup
R1(config)#ip domain-name cisco.com
R1(config)#username admin secret admin
R1(config)#ipv6 unicast-routing
R1(config)#crypto key generate rsa 
1024
R1(config)#ip ssh version 2

R1(config)#line console 0
R1(config-line)#password cisco
R1(config-line)#login
R1(config-line)#line vty 0 15
R1(config-line)#login local
R1(config-line)#transport input ssh

R1(config-line)#interface G0/0/0
R1(config-if)#ip address 192.168.1.1 255.255.255.0
R1(config-if)#description Connected to S1
R1(config-if)#ipv6 address 2001:DB8:ACAD:1::1/64
R1(config-if)#ipv6 address FE80::1 link-local
R1(config-if)#no shutdown

R1(config-line)#interface G0/0/1
R1(config-if)#ip address 192.168.2.1 255.255.255.0
R1(config-if)#description Connected to S1
R1(config-if)#ipv6 address 2001:DB8:ACAD:2::1/64
R1(config-if)#ipv6 address FE80::1 link-local
R1(config-if)#no shutdown

R1(config-if)#end
R1#copy running-config startup-config
