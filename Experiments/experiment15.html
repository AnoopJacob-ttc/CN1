Experiment – 15

CONFIGURE DHCP SERVER FOR MULTIPLE NETWORKS

← Back to Experiment List ← Back to Main Page

AIM

  - Connect and set basic configs on all devices
  - Assign different IPv4 networks to each LAN
  - Configure Server as DHCP server for all networks
  - Verify PCs get IPs automatically and test with ping

TOPOLOGY

ADDRESSING TABLE

Device Interface IPv4 Address Subnet Mask Default Gateway
R1 G0/0/0 192.168.1.1 255.255.255.0 NOT APPLICABLE
G0/0/1 192.168.2.1 255.255.255.0 NOT APPLICABLE
S1 VLAN 1 192.168.1.2 255.255.255.0 192.168.1.1
S2 VLAN 1 192.168.2.2 255.255.255.0 192.168.2.1
DHCP Server NIC 192.168.2.3 255.255.255.0 192.168.2.1
PC1 NIC DHCP
PC2 NIC DHCP
PC3 NIC DHCP

R1 CONFIGURATION

Router>enable
Router#clock set 10:30:15 01 January 2026
Router#configure terminal
Router(config)#no ip domain-lookup
Router(config)#hostname R1
R1(config)#service password-encryption
R1(config)#banner motd "Unauthorized access is strictly prohibited"
R1(config)#enable secret itsasecret

R1(config)#line console 0
R1(config-line)#password cisco
R1(config-line)#login
R1(config-line)#logging synchronous
R1(config-line)#exec-timeout 5
R1(config-line)#line vty 0 15
R1(config-line)#password cisco
R1(config-line)#login
R1(config-line)#logging synchronous
R1(config-line)#exec-timeout 5

R1(config-line)#interface g0/0/0
R1(config-if)#ip address 192.168.1.1 255.255.255.0
R1(config-if)#description Connected to S1
R1(config-if)#ip helper-address 192.168.2.3
R1(config-if)#no shutdown

R1(config-if)#interface g0/0/1
R1(config-if)#ip address 192.168.2.1 255.255.255.0
R1(config-if)#description Connected to S2
R1(config-if)#no shutdown
R1(config-if)#end

R1#copy running-config startup-config

S1 CONFIGURATION

Switch>enable
Switch#clock set 10:30:15 01 January 2026
Switch#configure terminal
Switch(config)#no ip domain-lookup
Switch(config)#hostname S1
S1(config)#service password-encryption
S1(config)#banner motd "Unauthorized access is strictly prohibited"
S1(config)#enable secret itsasecret

S1(config)#line console 0
S1(config-line)#password cisco
S1(config-line)#login
S1(config-line)#logging synchronous
S1(config-line)#exec-timeout 5
S1(config-line)#line vty 0 15
S1(config-line)#password cisco
S1(config-line)#login
S1(config-line)#logging synchronous
S1(config-line)#exec-timeout 5

S1(config-line)#interface vlan 1
S1(config-if)#ip address 192.168.1.2 255.255.255.0
S1(config-if)#exit
S1(config)#ip default-gateway 192.168.1.1
S1(config)#exit

S1#copy run start

CONFIGURE SERVER AS DHCP SERVER FOR ALL NETWORKS

Click on DHCP Server → Services → DHCP
Service → On

Pool Name: S1LAN
Default Gateway: 192.168.1.1
Start IP Address: 192.168.1.10
Subnet Mask: 255.255.255.0
Maximum Number of Users: 200
Click Add

Pool Name: serverPool
Default Gateway: 192.168.2.1
Start IP Address: 192.168.2.10
Subnet Mask: 255.255.255.0
Maximum Number of Users: 200
Click Add

PC1, PC2, AND PC3 CONFIGURATION

IP Configuration: DHCP

CONNECTIVITY TEST

Ping each PC and make sure that they communicate successfully with each other.

RESULT

The DHCP server was successfully configured to assign IP addresses to multiple networks, and all PCs received IPs automatically and communicated successfully using ping.
