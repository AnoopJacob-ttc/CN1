Experiment – 14

CONFIGURE SECURE PASSWORDS, SSH, AND TELNET ON A CISCO ROUTER

← Back to Experiment List ← Back to Main Page

AIM

  - Configure secure Password
  - Configure SSH & Telnet
  - Verify SSH & Telnet

TOPOLOGY

ADDRESSING TABLE

Device Interface IP Address Subnet Mask Default Gateway Description
R1 G0/0/0 192.168.1.1 255.255.255.0 NOT APPLICABLE Connected to S1 G0/1
S1 VLAN 1 192.168.1.2 255.255.255.0 192.168.1.1 SVI for Management
PCA NIC 192.168.1.10 255.255.255.0 192.168.1.1 Connected to S1 Fa0/1

R1 CONFIGURATION (SSH)

Router>
Router>enable
Router#conf t
Router(config)#hostname R1
R1(config)#enable secret class
R1(config)#service password-encryption
R1(config)#banner motd "Warning! Authorized access only"
R1(config)#no ip domain-lookup
R1(config)#ip domain-name cisco.com
R1(config)#username admin secret admin
R1(config)#crypto key generate rsa 
1024
R1(config)#ip ssh version 2

R1(config)#line console 0
R1(config-line)#password cisco
R1(config-line)#login
R1(config-line)#line vty 0 15
R1(config-line)#login local
R1(config-line)#transport input ssh

R1(config-line)#interface g0/0/0
R1(config-if)#ip address 192.168.1.1 255.255.255.0
R1(config-if)#no shutdown
R1(config-if)#end

R1#copy run start

S1 CONFIGURATION (TELNET)

Switch>enable
Switch#conf t
Switch(config)#hostname S1
S1(config)#enable secret class
S1(config)#service password-encryption
S1(config)#banner motd "Warning! Authorized access only"
S1(config)#no ip domain-lookup
S1(config)#ip default-gateway 192.168.1.1

S1(config)#line console 0
S1(config-line)#password cisco
S1(config-line)#login
S1(config-line)#line vty 0 15
S1(config-line)#password cisco
S1(config-line)#login

S1(config-line)#interface vlan 1
S1(config-if)#ip address 192.168.1.2 255.255.255.0
S1(config-if)#no shutdown
S1(config-if)#end

S1#copy run start

PCA IP CONFIGURATION

IPv4 Address: 192.168.1.10
Subnet Mask: 255.255.255.0
Default-Gateway: 192.168.1.1

ACCESS THE ROUTER USING SSH THROUGH THE PC-A COMMAND PROMPT

C:\>ssh -l admin 192.168.1.1

ACCESS THE SWITCH USING TELNET THROUGH THE PC-A COMMAND PROMPT

C:\>telnet 192.168.1.1

SHOW COMMANDS USED FOR VERIFICATION

R1#show running-config
R1#show ip interface brief
S1#show ip interface brief

RESULT

Accessed Router using SSH and Switch using Telnet from PCA
