Experiment – 11

CONNECT A ROUTER TO A LAN USING IPV4 ADDRESSING

← Back to Experiment List ← Back to Main Page

AIM

  - Display Router Information
  - Configure Basic Settings
  - Configure Router Interfaces
  - Verify the Configuration

TOPOLOGY

ADDRESSING TABLE

Device Interface IP Address Subnet Mask Default Gateway
R1 G0/0/0 192.168.1.1 255.255.255.0 NOT APPLICABLE
G0/0/1 192.168.2.1 255.255.255.0 NOT APPLICABLE
S1 VLAN 1 192.168.1.2 255.255.255.0 192.168.1.1
S2 VLAN 1 192.168.2.2 255.255.255.0 192.168.2.1
PC1 NIC 192.168.1.10 255.255.255.0 192.168.1.1
PC2 NIC 192.168.1.11 255.255.255.0 192.168.1.1
PC3 NIC 192.168.2.10 255.255.255.0 192.168.2.1
PC4 NIC 192.168.2.11 255.255.255.0 192.168.2.1

DISPLAY ROUTER INFORMATION

PERFORM BASIC CONFIGURATIONS ON S1

Router>enable
Router#show running-config

CONFIGURE BASIC SETTINGS

Router>enable
Router#clock set 10:30:15 01 January 2026
Router#configure terminal
Router(config)#no ip domain-lookup
Router(config)#hostname R1
R1(config)#banner motd "Unauthorized access is strictly prohibited"
R1(config)# ip domain-name nttftrg.com
R1(config)#enable password cisco
R1(config)#enable secret itsasecret
R1(config)#service password-encryption

R1(config)#line console 0
R1(config-line)#password cisco
R1(config-line)#login
R1(config-line)# logging synchronous
R1(config-line)#exec-timeout 5

R1(config-line)#line vty 0 15
R1(config-line)#password cisco
R1(config-line)#login
R1(config-line)# logging synchronous
R1(config-line)#exec-timeout 5
R1(config-line)#exit

CONFIGURE ROUTER INTERFACES

R1(config)#interface G0/0/0
R1(config-if)#ip address 192.168.1.1 255.255.255.0
R1(config-if)#no shutdown

R1(config-if)#interface G0/0/1
R1(config-if)#ip address 192.168.2.1 255.255.255.0
R1(config-if)#no shutdown

R1(config-if)#end
R1#copy running-config startup-config

S1 CONFIGURATION

Switch>enable
Switch#configure terminal
Switch(config)#hostname S1

S1(config)#interface vlan1
S1(config-if)#ip address 192.168.1.2 255.255.255.0
S1(config-if)#no shutdown
S1(config-if)#exit

S1(config)#ip default-gateway 192.168.1.1
S1(config)#exit

S1#copy running-config startup-config

S2 CONFIGURATION

Switch>enable
Switch#configure terminal
Switch(config)#hostname S2

S2(config)#interface vlan1
S2(config-if)#ip address 192.168.2.2 255.255.255.0
S2(config-if)#no shutdown
S2(config-if)#exit

S2(config)#ip default-gateway 192.168.2.1
S2(config)#exit

S2#copy running-config startup-config

CONFIGURE THE PCs

PC1 IP CONFIGURATION

IPv4 Address: 192.168.1.10
Subnet Mask: 255.255.255.0
Default Gateway: 192.168.1.1

PC2 IP CONFIGURATION

IPv4 Address: 192.168.1.11
Subnet Mask: 255.255.255.0
Default Gateway: 192.168.1.1

PC3 IP CONFIGURATION

IPv4 Address: 192.168.2.10
Subnet Mask: 255.255.255.0
Default Gateway: 192.168.2.1

PC4 IP CONFIGURATION

IPv4 Address: 192.168.2.11
Subnet Mask: 255.255.255.0
Default Gateway: 192.168.2.1

SHOW COMMANDS USED FOR VERIFICATION

R1#show running-config
R1#show ip interface brief
S1#show ip interface brief
S2#show ip interface brief
C:/>ipconfig /all

RESULT:

Configured and verified Connect a Router to LAN
